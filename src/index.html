Democratizando o ensino

Biblioteca de vídeo aulas gratuita para indexar conteúdo descentralizado. Existe muito conteúdo gratuito no youtube, mas nem sempre com o mesmo canal ou professor. Sem falar que o algoritmo do Youtube desfavorece os canais pequenos.

Quando se faz uma busca no Youtube, o google retorna poucos vídeos e logo na sequência mostra uma sessão como: "Para você". E essa sessão na maioria das vezes não tem relação com a busca original, mas sim com vídeos que o usuário já assistiu. O google está empenhado em manter as pessoas em bolhas, dificultando cada vez mais o acesso aos conteúdos educacionais.

O banco de dados guarda somente as urls dos vídeos. Eles ainda estarão hospedados no youtube, dando a visualização e a monetização para o criador original. Além disso, criadores de conteúdos com pouco alcance podem entrar em contato e pedir para que as suas aulas sejam adicionadas a plataforma.

Pokedex

Aplicação web de uma Pokedex. Serve para listar e buscar pelos monstrinhos. A aplicação tem seu frontend e backend separados e eu planejo fazer um aplicativo mobile que também acessa a api.

O front foi feito com Typescript, React e Vite. E o deploy foi feito no Netlify. São basicamente duas páginas. Uma para listar ou pesquisar os pokemons e outra com os dados individuais de cada um.

A API foi feita com Node.js, Typescript, Nest, Prisma e MongoDB. Eu só a criei porque estava insatisfeito com a PokeApi. O problema da PokeApi está na listagem de pokemons, que só retorna uma url e um nome. Ela só retorna as informações do pokemon quando se busca especificamente por ele. Para essa aplicação eu precisava de todas as informações já na listagem. Autenticação não é necessária.

Biblioteca

Um CRUD de biblioteca com backend e frontend separados. O backend usa Node.js, Typescript, Express.js e o banco de dados Firestore. O frontend usa Typescript, React e vite. Features:

  • Testes unitários e de integração na api.
  • Autenticação implementada no front e no backend.
  • Validação de dados no client e no server.
  • O banco de dados está hospedado no google.
  • O front está hospedado no netlif.
